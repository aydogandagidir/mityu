# Mityu v1.0.4 release candidate

**Status (2026-07-15):** The Windows x64 implementation candidate passes the completed local engineering gates and is tracked by release PR #4. Protected exact-SHA CI passed on the implementation commit and must remain green on every later PR head before merge. ADR-0027 makes the still-unperformed A5 real-audio evaluation and C8 human pilot deferred, non-blocking evidence debt for v1.0.4 only; neither is PASS. Publication remains **NO-GO** until landing-data disposition and authorised legal/commercial gates below are closed. Windows signing scope is resolved: v1.0.4 is updater-signed only, Authenticode deferred (ADR-0029). After publication, broad rollout/announcement remains **NO-GO** until the installed-v1.0.3 updater canary passes.

## Scope

v1.0.4 is based on the published v1.0.3 line and keeps its production platform scope: Windows x64. The published v1.0.3 tag is immutable. macOS is excluded from this release because its FFmpeg provenance, signing/notarization and physical smoke-test gates are not complete.

**Validation limitation:** target-environment WER/diarization validation and a signed human pilot were deferred for v1.0.4. Accuracy varies by language, speaker, microphone, overlap and environment. Users must verify transcripts and AI-generated drafts against the source audio/transcript; this release makes no field-suitability or accuracy-SLA claim.

The candidate adds:

- Ranked, offline Evidence Search over active transcript segments, with exact source timestamps and deep links.
- An Approved Action Center that shows only human-approved, active, source-resolvable action items.
- Stronger draft/approval provenance, same-tenant source validation and optimistic revision checks.
- A single application-version source for browser surfaces and synchronized package/Tauri/Rust versions.
- A fail-closed Windows release workflow: same-SHA CI, strict SemVer, updater-signing preflight, immutable third-party Action SHAs, exact asset/updater-manifest checks and baked-key updater verification. Windows Authenticode is deferred for v1.0.4 (ADR-0029): the workflow ships **updater-signed only**, matching the published v1.0.0–v1.0.3 line, so the Authenticode preflight/verification is intentionally not exercised.
- Pinned, SHA-256-verified LGPL FFmpeg acquisition with no runtime executable download fallback.
- Verifiable cleanup of Mityu-managed database, FTS, WAL, recording and recovery-cache data, with explicit storage-layer limitations.
- Context-bound recording completion: one native stop owner, opaque one-use save tokens, Rust-only folder authority, idempotent save/ack retry and reload-safe recovery without cross-recording folder reuse.
- Removal of the unsupported psychiatric-session template; v1.0.4 is not validated or positioned for clinical documentation or medical decision-making.

## Proposed public release notes

### Mityu v1.0.4

This release makes reviewed meeting knowledge easier to find and act on while keeping processing local by default.

Target-environment accuracy and diarization validation plus a signed human pilot were deferred for this patch. Transcription quality varies by language, speaker, microphone, overlap and environment; verify transcripts and AI drafts against their source evidence.

Raw meeting audio remains on the user's device until the meeting is deleted; v1.0.4 does not automatically delete it immediately after transcription.

- Search transcript evidence across meetings with ranked results, source timestamps and direct jumps to the exact segment.
- Review human-approved follow-ups in one Action Center, with assignee, due text and source evidence.
- Improved provenance and approval safeguards so AI-generated content stays draft-only until a person approves it.
- Removed an unsupported psychiatric-session template; this release is not validated for clinical documentation or medical decision-making.
- Meeting deletion now verifies cleanup across Mityu-managed local database, search, recording and recovery stores; storage snapshots, backups, exports and device-level physical remnants remain outside the app's control.
- Hardened offline packaging, dependency verification and Windows update integrity.

Important: the new transcript search index is a forward-only database migration. Downgrading the application after opening the database with v1.0.4 is not supported.
